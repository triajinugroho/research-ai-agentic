## Tujuan Pembelajaran

Setelah menyelesaikan modul ini, mahasiswa diharapkan mampu:

1. **Mempresentasikan** proyek perangkat lunak secara profesional dengan demo langsung (C6)
2. **Mengevaluasi** proyek tim lain berdasarkan kriteria teknis dan proses (C5)
3. **Memberikan** feedback konstruktif melalui peer review (C5)

---

## Materi Pembelajaran

### 15.1 Format Demo Day

Setiap tim memiliki **15 menit** total:

| Waktu | Aktivitas |
|-------|-----------|
| 0-2 menit | Perkenalan tim dan overview proyek |
| 2-8 menit | **Live demo** aplikasi — tunjukkan fitur utama dan user flow |
| 8-11 menit | Technical highlights: arsitektur, teknologi, CI/CD pipeline, AI usage |
| 11-13 menit | Lessons learned dan refleksi proses Agile/Scrum |
| 13-15 menit | Q&A dari dosen dan mahasiswa |

### 15.2 Panduan Presentasi

#### Yang Harus Ditunjukkan

1. **Aplikasi berjalan** (live demo, bukan screenshot) — deploy ke cloud jika memungkinkan
2. **Repository GitHub** — commit history, branching strategy, PR reviews
3. **CI/CD pipeline** — tunjukkan workflow GitHub Actions berjalan
4. **Test suite** — jalankan test dan tunjukkan coverage
5. **AI Usage Log** — transparansi penggunaan AI dalam pengembangan

#### Tips Presentasi

- Siapkan **backup plan** (video rekaman demo) jika live demo gagal
- Fokus pada **value** yang dihasilkan, bukan hanya fitur teknis
- Gunakan **skenario nyata** untuk demo, bukan data dummy
- Setiap anggota tim harus berbicara minimal satu bagian

### 15.3 Rubrik Penilaian Proyek Akhir

| Kriteria | Bobot | Deskripsi |
|----------|-------|-----------|
| **Fungsionalitas** | 25% | Fitur berjalan sesuai requirements, UX baik |
| **Kualitas Kode** | 20% | Clean code, design patterns, test coverage ≥ 70% |
| **Proses SE** | 20% | Agile/Scrum, Git flow, code review, CI/CD |
| **Presentasi** | 15% | Komunikasi jelas, demo lancar, Q&A solid |
| **AI Integration** | 10% | Penggunaan AI yang efektif dan bertanggung jawab |
| **Dokumentasi** | 10% | README, API docs, AI Usage Log lengkap |
| **Total** | **100%** | |

## Kegiatan Pembelajaran

### Pre-class (15 menit)

- Finalisasi slide presentasi dan persiapan demo
- Pastikan aplikasi berjalan di environment demo (cloud deployment)
- Siapkan backup video demo

### In-class (120 menit)

| Waktu | Aktivitas | Metode |
|-------|-----------|--------|
| 0-5 menit | Penjelasan format dan rubrik | Briefing |
| 5-95 menit | Presentasi tim (6 tim × 15 menit) | Presentasi + Q&A |
| 95-110 menit | Pengisian peer review form | Individual |
| 110-120 menit | Closing remarks, persiapan UAS Minggu 16 | Diskusi kelas |

### Post-class (15 menit)

- Submit peer review form melalui LMS
- Perbaiki proyek berdasarkan feedback (opsional, untuk portfolio)
- Mulai review materi Minggu 1-14 untuk persiapan UAS
