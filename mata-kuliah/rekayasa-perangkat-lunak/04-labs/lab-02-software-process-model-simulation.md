| **Platform** | GitHub Codespaces |

## Tujuan

1. **Membuat** Product Backlog menggunakan GitHub Issues
2. **Mensimulasikan** sprint planning, daily standup, sprint review, dan retrospective
3. **Menggunakan** GitHub Projects sebagai Scrum board

## Persiapan

- Repository GitHub dari Lab 01
- Akses GitHub Projects

## Langkah-langkah

### Langkah 1: Buat Product Backlog (20 menit)
Buat 8 GitHub Issues sebagai backlog items:
1. `feat: halaman login` — Must Have
2. `feat: halaman register` — Must Have
3. `feat: pencarian buku` — Must Have
4. `feat: detail buku` — Must Have
5. `feat: peminjaman buku` — Must Have
6. `feat: daftar peminjaman saya` — Should Have
7. `feat: notifikasi deadline` — Should Have
8. `feat: dark mode` — Could Have

Untuk setiap issue, tambahkan label (priority, type) dan deskripsi.

### Langkah 2: Setup GitHub Projects Board (15 menit)
1. Repository → Projects → New Project → Board
2. Kolom: `Backlog` | `Sprint Backlog` | `In Progress` | `Review` | `Done`
3. Drag semua issues ke kolom Backlog

### Langkah 3: Sprint Planning (15 menit)
1. Pilih Sprint Goal: "User bisa login dan mencari buku"
2. Drag 4 items Must Have ke Sprint Backlog
3. Assign anggota tim ke setiap item
4. Tuliskan Sprint Goal di comment issue pertama

### Langkah 4: Simulasi Daily Standup (15 menit)
Setiap anggota menjawab 3 pertanyaan (tulis di shared doc):
- Kemarin saya mengerjakan: ___
- Hari ini saya akan mengerjakan: ___
- Hambatan: ___

Pindahkan 2 item ke "In Progress".

### Langkah 5: Sprint Review & Retrospective (20 menit)
**Sprint Review:** Pindahkan 2-3 item ke Done. Diskusikan: apa yang berhasil di-deliver?

**Retrospective (Start-Stop-Continue):**
- **Start:** ___
- **Stop:** ___
- **Continue:** ___

## Tantangan Tambahan

1. Tambahkan story points (label: 1, 2, 3, 5, 8) ke setiap issue
2. Buat milestone "Sprint 1" dan assign issues yang relevan
3. Hitung velocity (total points Done) dan prediksi Sprint 2

## Checklist Penyelesaian

- [ ] 8 GitHub Issues dibuat dengan label dan deskripsi
- [ ] GitHub Projects board dengan 5 kolom
- [ ] Sprint planning tercatat (sprint goal + sprint backlog)
- [ ] Daily standup notes tercatat
- [ ] Sprint review + retrospective tercatat
