## Contoh Soal

### Bagian A: Pilihan Ganda (C1-C2)

1. Manakah yang BUKAN merupakan Scrum Event?
   - a) Sprint Planning  b) Daily Standup  c) Code Review  d) Sprint Retrospective

2. Prinsip SOLID yang menyatakan "class hanya boleh punya satu alasan untuk berubah" adalah:
   - a) Open/Closed  b) Single Responsibility  c) Liskov Substitution  d) Interface Segregation

3. Dalam MoSCoW prioritization, "Should have" berarti:
   - a) Wajib ada  b) Penting tapi tidak kritis  c) Nice to have  d) Tidak untuk rilis ini

### Bagian B: Uraian Singkat (C2-C3)

4. Jelaskan perbedaan Functional Requirements dan Non-Functional Requirements. Berikan masing-masing 2 contoh untuk Sistem Perpustakaan Kampus. (6 poin)

5. Sebutkan dan jelaskan 6 kriteria INVEST dalam menulis User Stories. (6 poin)

### Bagian C: Studi Kasus (C3-C4)

6. Sebuah rumah sakit di Jakarta ingin membangun Sistem Informasi Rekam Medis. Tim 8 developer, deadline 8 bulan, requirements dari Kemenkes (relatif stabil, regulasi ketat). (25 poin)
   - a) Model proses apa yang paling cocok? Jelaskan alasan. (5 poin)
   - b) Buatlah 5 user stories untuk fitur utama. (5 poin)
   - c) Gambarkan Class Diagram dengan minimal 4 class dan relasi. (8 poin)
   - d) Identifikasi 3 code smells dalam kode berikut dan jelaskan refactoring-nya. (7 poin)

## Petunjuk Belajar

- Review materi Minggu 1-7 (modul + buku ajar Bab 1-7)
- Latihan soal di akhir setiap bab buku ajar
- Fokus pada **pemahaman konsep** (C2) dan **kemampuan menerapkan** (C3-C4)
- Pahami diagram (UML, ERD) dan bisa menggambarnya
