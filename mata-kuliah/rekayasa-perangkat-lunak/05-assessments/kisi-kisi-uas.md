## Contoh Soal

### Bagian A: Pilihan Ganda (C1-C3)

1. Dalam testing pyramid, testing mana yang paling banyak jumlahnya?
   - a) E2E Testing  b) Integration Testing  c) Unit Testing  d) Acceptance Testing

2. Perintah Docker yang digunakan untuk menjalankan multi-container setup adalah:
   - a) `docker run`  b) `docker compose up`  c) `docker build`  d) `docker exec`

3. Semantic Versioning 2.0.0 → 2.1.0 menandakan:
   - a) Breaking change  b) Bug fix  c) Fitur baru backward-compatible  d) Security patch

### Bagian B: Uraian (C2-C4)

4. Jelaskan siklus TDD (Red-Green-Refactor) dan berikan contoh penerapannya untuk fitur "pencarian buku". (5 poin)

5. Jelaskan 3 keterbatasan AI dalam software engineering dan bagaimana seorang engineer harus mengatasinya. (5 poin)

### Bagian C: Studi Kasus (C4-C6)

6. Sebuah startup fintech membangun aplikasi payment gateway. Tim 6 developer, sudah berjalan 6 bulan, menggunakan Flask + PostgreSQL + Docker. (18 poin)
   - a) Rancang CI/CD pipeline menggunakan GitHub Actions — jelaskan setiap step. (6 poin)
   - b) Buatlah Dockerfile dan jelaskan setiap instruksi. (4 poin)
   - c) Rancang strategi testing komprehensif (unit, integration, E2E, security). (5 poin)
   - d) Identifikasi 3 risiko teknis dan mitigasinya. (3 poin)

7. Anda diminta mengevaluasi penggunaan AI (Copilot/Claude) dalam proyek tim Anda. (18 poin)
   - a) Di fase SDLC mana AI paling membantu? Berikan evidence dari pengalaman proyek. (6 poin)
   - b) Sebutkan 3 kasus di mana AI output perlu dikoreksi dan jelaskan mengapa. (6 poin)
   - c) Bagaimana prinsip amanah dan kejujuran akademik diterapkan dalam penggunaan AI? (6 poin)
